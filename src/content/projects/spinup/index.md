SpinUp is an imagined developer tool that turns a multi-day onboarding into a single
14-second command — automating `.env` configuration, dependency installs, Prisma
migrations, and database seeding.

The product doesn't exist. The point of the piece is to demonstrate, in one cohesive
artifact, the two things I do: **write direct-response B2B SaaS copy for a technical
audience**, and **build the fast, responsive front end that delivers it**.

## The copy

Punchy, logical, no marketing fluff — aimed at Lead Developers, Engineering Managers,
and CTOs. The page is built around a clear problem → agitation → solution → proof → CTA
structure, with a voice that respects a technical reader's time.

## The build

A fully responsive, dark-mode landing page built from scratch in React, with custom
animations and a strict, self-imposed design system:

- An animated before/after terminal that types out the painful manual setup versus the
  one-command SpinUp flow, with a replay control.
- A syntax-highlighted config block with a working copy-to-clipboard button.
- Scroll-reveal animations that respect `prefers-reduced-motion`.
- A programmatically generated Open Graph share card.
- A disciplined system: hairline borders, a single accent color, two intentional glows,
  and zero gratuitous shadows or gradients.

> SpinUp is a fictional product created for portfolio purposes. It is not a real service
